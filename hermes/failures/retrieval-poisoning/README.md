# Failure Class: Retrieval & Memory Poisoning

This directory codifies the failure postures, containment mechanisms, and recovery procedures for memory drift, quarantine pressure, and secret leaks.

---

## F4. Memory drift and Ingestion Poisoning

*   **Class:** `DEGRADED` to `CRITICAL`
*   **Symptom:** A periodic Atlas contradiction sweep finds a high rate of conflicting facts, or untrusted source ingestion attempts to force promotions without corroboration.
*   **Containment:** The **Quarantine Tier (I6)** prevents uncorroborated facts from entering authoritative Atlas retrieval. Contradicting facts are logged with source signatures rather than overwritten.
*   **Recovery:**
    1. Requantize the affected facts back to the quarantine tier.
    2. Halt automatic ingestion from the flagged source path.
    3. Alert the operator to audit the Git/external files for active prompt injection payloads.
*   **Telemetry:** `atlas.drift_score`, `atlas.quarantine_depth`.

---

## F5. Quarantine Pressure Overflow

*   **Class:** `DEGRADED`
*   **Symptom:** The quarantine memory storage exceeds its maximum depth limit due to an aggressive stream of untrusted fact ingestion.
*   **Why concerning:** This indicates a deliberate attempt to poison memory or trigger a buffer/memory-based exhaustion attack.
*   **Containment:** Atlas enforces an aggressive TTL on quarantine entries; oldest unpromoted facts are automatically discarded once size ceilings are breached.
*   **Recovery:**
    1. Throttle the ingestion rate on the active session.
    2. Require operator re-authentication or source review.
*   **Telemetry:** `atlas.quarantine_growth_rate`.

---

## F6. Secret Leakage to Model Context (Canary Alert)

*   **Class:** `CATASTROPHIC`
*   **Symptom:** A plaintext credential or synthetic vault canary secret is detected in an outbound LLM prompt, trace log, or telemetry payload.
*   **Why concerning:** A secret has escaped the Vault boundaries, risking immediate exposure to external model providers or logs.
*   **Containment:** The **Entropy/Regex Canary Scanner (I3)** immediately blocks the outbound transmission at the network interface before it is written or sent.
*   **Recovery:**
    1. Immediately quarantine the active session.
    2. Force-revoke all capability tokens generated by the session KEK.
    3. Trigger immediate key rotation for the leaked credential in Vault.
    4. Audit all API access logs touching the affected credential scope since last known-good state.
*   **Telemetry:** `secrets.canary_detected_in_outbound` (triggers immediate CATASTROPHIC severity alarms and pager alerts).
